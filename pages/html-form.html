Attributes

  • <form ... novalidate> - to avoid validation by browser of a specific fields
  • <form ... autocomplete="on"> ... - autocomplete with previously inserted values, forms inputs
  • <input autocomplete="off"> ... - disable some inputs autocompletion
  • <input type="image" src="img.gif" alt="Submit" width="48" height="48"> - define an image as the submit button
  • Regular
    • disabled - field should be disabled
    • readonly - read only (cannot be changed)
    • maxlength - maximum number of characters
    • size - width (in characters)
    • value - default value
  • HTML5
    • required - required (must be filled out) unsupported in IE9 (and earlier), or in Safari
    • min - minimum value unsupported in IE9 (and earlier), and IE10 for dates and time
    • max - maximum value unsupported in IE9 (and earlier), and IE10 for dates and time
    • step - legal number intervals unsupported in IE9 (and earlier)
    • pattern[="[A-Za-z]{3}"] - regular expression to check the input value against unsupported in IE9 (and earlier), or in Safari 10 and earlier
    • placeholder[="First name"] - a hint that describes the expected value unsupported in IE
    • autofocus - field should automatically get focus unsupported in IE9 (and earlier)
    • multiple - user is allowed to enter more than one value in the input element (used with email and file buttons)unsupported in IE9 (and earlier)
    • form - specifies one or more forms an element belongs to unsupported in IE
    • formaction[="/action_page2.php"] - overrides the action attribute of the form (used with submit and image buttons) unsupported in IE9 (and earlier)
    • formenctype[="multipart/form-data"] - how the form data should be encoded when submitted (used with submit and image buttons, only for forms with method="post") unsupported in IE9 (and earlier)
    • formmethod[="post"] - overrides the method attribute of the form-element (used with submit and image buttons) unsupported in IE9 (and earlier)
    • formnovalidate - overrides the method attribute of the form-element (used with submit buttons) unsupported in IE9 (and earlier), or in Safari 10 and earlier
    • formtarget[="_blank"] - overrides the target attribute of the form-element (used with submit and image buttons) unsupported in IE9 (and earlier)
    • minlength - minimum number of characters
    • autocapitalizealue - capitalization behavior for user input: none, off, characters, words, sentences
